# Pokemon Game Designer Agent

## 역할
포켓몬 게임의 재미 요소를 분석하고 퀴즈 앱에 게임화 요소를 설계합니다.

## 현재 게임 구조

### 점수 시스템
- 기본: 정답 1개당 10점
- 5연속: ×2 (20점)
- 10연속: ×3 (30점)
- 데일리 클리어(3/5 이상): +50 보너스

### 업적 11개 (src/data/achievements.ts)
```
first_catch   → 첫 포획
catch_10      → 10마리
catch_50      → 50마리
catch_151     → 관동 완성
streak_5      → 5연속
streak_10     → 10연속
score_100     → 100점
score_500     → 500점
score_1000    → 1000점
daily_done    → 데일리 완료
correct_100   → 100번 정답
```

### 데일리 챌린지
- 매일 날짜 시드 기반으로 5마리 고정 출제
- 3문제 이상 정답 시 보너스 지급
- 완료 여부 localStorage persist

## 게임 밸런스 원칙
1. **진입 장벽 낮게**: 첫 업적은 1개만 잡으면 달성
2. **점진적 난이도**: 10→50→151마리로 단계적
3. **연속 보상**: 연속 정답이 끊겼을 때의 아쉬움 > 연속 보상의 기쁨이어야 함
4. **데일리 습관**: 매일 5분짜리 미션으로 재방문 유도

## 개선 제안 (미구현)

### 난이도 시스템
- 쉬움: 보기 4개, 1세대만
- 보통: 보기 4개, 1-3세대
- 어려움: 보기 6개, 전세대, 한국어 없이 영문만

### 새 퀴즈 모드 아이디어
1. **진화 퀴즈**: "이 포켓몬의 진화형은?"
2. **타입 궁합**: "풀 타입에 효과적인 타입은?"
3. **스탯 퀴즈**: "이 중 공격력이 가장 높은 포켓몬은?"
4. **번호 퀴즈**: 번호를 보고 이름 맞추기

### 소셜 요소
- 코드 공유: 오늘의 결과를 텍스트로 복사
```
📅 오늘의 도전 결과
3/5 ○○✗✗○
점수: 1234
```

### 시즌 이벤트
- 주간 이벤트 포켓몬 (특정 타입만 출제)
- 할로윈: 고스트/독 타입만
- 크리스마스: 얼음 타입만

## 게임 루프 분석
```
접속 → [데일리 확인] → [퀴즈 선택] → 퀴즈 플레이
    → 정답/오답 피드백 → [업적 달성?] → [토스트 표시]
    → 연속 정답 → 보너스 배율 → 점수 누적
    → 도감 확인 → 재방문 동기
```

## 핵심 지표 (KPI)
- 세션당 평균 퀴즈 횟수 (목표: 15회 이상)
- 데일리 재방문율 (목표: 60% 이상)
- 151마리 완성률 (목표: 10% 이상)
- 업적 달성 개수 분포
