# Pokemon Testing Skill

## 테스트 환경 설정
```bash
cd pokemon-app
npm run dev     # http://localhost:5173
```
DevTools: F12 → Application → Local Storage에서 상태 확인 가능

## 핵심 테스트 시나리오

### 시나리오 1: 신규 사용자 전체 플로우
```
1. http://localhost:5173 접속
2. "트레이너 등록" 클릭 → /login 이동
3. 닉네임 입력: "테스트123"
4. 트레이너 아바타 선택 (2번째)
5. 지방 선택: "호연"
6. "시작!" 클릭
7. ✅ 홈에서 "테스트123 트레이너, 어서오세요!" 표시 확인
8. 실루엣 퀴즈 클릭
9. ✅ 그림자 보임 (검정 silhouette)
10. 정답 클릭
11. ✅ 이펙트 + "정답! [이름]" 표시
12. ✅ 나의 도감에서 포획 수 1 증가
```

### 시나리오 2: 업적 달성 테스트
```
LocalStorage 조작:
  pokemon-game-store → capturedPokemon: [1,2,...,9] (9개)
1. 실루엣 퀴즈에서 1문제 더 정답
2. ✅ "포켓몬 탐험가" 토스트 팝업 (4초 후 사라짐)
3. ✅ 나의 도감 업적 탭에서 달성 표시
```

### 시나리오 3: 연속 정답 배율
```
1. 실루엣 퀴즈 진입
2. 4번 연속 정답
3. ✅ 배율 배너 없음 (5연속 아직)
4. 5번째 정답
5. ✅ "연속 정답: 5 🔥🔥 2배!" 표시
6. ✅ 점수 +20 (10×2)
7. 10번째 정답
8. ✅ "x3" 배율 표시
```

### 시나리오 4: 데일리 챌린지
```
1. 홈에서 오늘의 도전 배너 클릭
2. ✅ 날짜 표시, 1/5 진행 표시
3. 3문제 틀림 + 2문제 맞춤
4. ✅ 결과 화면: 2/5, 보너스 없음
홈으로 돌아가기 클릭
1. ✅ 오늘의 도전 배너 사라짐
2. ✅ 재도전 불가 ("오늘의 도전 완료!" 화면)
```

### 시나리오 5: 로그아웃 & 재로그인
```
1. NavBar 트레이너 이름 클릭
2. "로그아웃" 클릭
3. ✅ 홈에 "트레이너 등록" 버튼 표시
4. 퀴즈 클릭 → /login 리다이렉트
5. 다시 등록
6. ✅ 기존 점수/포획 유지 (gameStore는 별도 persist)
```

## 엣지 케이스

### PokeAPI 에러 처리
```
DevTools Network 탭 → pokeapi.co 요청 막기 (Block request URL)
1. 실루엣 퀴즈 접속
2. ✅ 로딩 스피너 표시 (무한 로딩 - 현재 에러 UI 없음)
개선 필요: 에러 상태 UI 추가
```

### 빠른 연속 클릭
```
1. 퀴즈 화면에서 정답 클릭
2. 즉시 다른 선택지 클릭
3. ✅ 중복 처리 안 됨 (disabled 처리)
```

### 모바일 375px 테스트
```
DevTools → 375px 뷰포트
1. NavBar 오버플로우 없음?
2. 퀴즈 선택지 2×2 그리드?
3. 버튼 터치 영역 충분?
4. 텍스트 잘림 없음?
```

## 알려진 제한사항
- PokeAPI rate limit: 과도한 요청 시 429 에러 (개발 환경)
- 이미지 로딩: 느린 네트워크에서 blank → spinner → 이미지 순서
- 점수는 localStorage persist - 브라우저 캐시 삭제 시 초기화
- 트레이너와 게임 데이터가 별도 키에 저장됨 (의도적)
